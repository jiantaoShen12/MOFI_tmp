你这个页面现在的主要问题不是“绿色不好”或“黄色不好”，而是**UI 装饰色、数据可视化色、背景底色、强调文字色混在同一个层级里使用**。例如 CSS 里背景是偏暖黄的 `--bg: #fbfaf6`，强调色是蓝紫 `--accent: #5c82dc`，生物感绿是 `--mint: #6bd39b`；同时 hero 泡泡、kicker、标签、标题渐变又大量用绿/青色，JS 里的轨迹色还包含高饱和黄色 `#FFC829` 和绿色 `#7ABC3C`。这些颜色都各自好看，但放在同一个页面层级里会显得散、乱、廉价。

## 一、总体设计原则：不要“绿色/黄色混搭”，而是建立三层色彩系统

我建议把页面改成 **“科研型暖白底 + 深蓝墨色文字 + 青蓝主色 + 少量生物绿 + 极少量琥珀黄”**。

具体逻辑是：

**背景色**只负责“干净、温润、像论文/实验室页面”，不要抢内容。黄色不能是明显的黄，而应该变成很浅的 warm ivory / culture-media tint，比如 `#fbfaf3`、`#f7f3e8`、`#fffaf0` 这一类。

**UI 主色**用青蓝或蓝紫，负责按钮、导航、交互状态。现在 `--accent: #5c82dc` 可以保留，但要把它升级成一套系统，比如主色 `#3f6fd8`，深色 `#203a75`，浅色 `#eaf1ff`。

**生物感绿色**只作为“生命、细胞、growth、proliferation”的轻量辅助，不要让绿色承担所有标题、标签、边框、背景。现在 `--mint: #6bd39b` 可保留，但饱和度要降，比如用 `#7ccf9a` 或 `#9edbb8`，并且主要用在小圆点、微弱光晕、少数模块角标。

**黄色/琥珀色**只作为数据语义或局部高亮，不要作为大面积背景。现在轨迹和 TF legend 中有明显黄橙色渐变，黄色已经有“数据时间/强度”的语义，所以页面背景再黄，就会和图例、轨迹冲突。

## 二、推荐的新视觉风格：Bio-Computational Editorial

我建议用这个方向：

> **Bio-Computational Editorial / Nature Methods-style scientific interface**
> 温润浅底，深蓝文字，青蓝主色，低饱和生物绿，少量琥珀表示时间/能量/扰动。

整体观感应该像：**干净的高水平计算生物学方法网页，而不是彩色 dashboard**。

### 推荐色板

```css
:root {

  --bg-soft: #f5f8f7;         /* 淡实验室冷白 */
  --paper: rgba(255,255,255,.88);
  --paper-solid: #ffffff;

  --muted: #66758a;
  --muted-2: #8a98aa;

  /* line/shadow */
  --line: rgba(23,32,51,.09);
  --line-strong: rgba(23,32,51,.14);
  --shadow-soft: 0 18px 55px rgba(28,55,84,.10);
  --shadow-card: 0 16px 42px rgba(28,55,84,.12);

  /* UI accent */

  --accent-deep: #203a75;
```

这套方案会解决两个核心问题：
第一，背景虽然保留温暖感，但不会再像“黄色底板”；第二，绿色从主视觉退到生物辅助色，页面会更像科研方法网站。

## 三、页面布局层级怎么改

你现在 HTML 的结构其实是合理的：顶部导航、hero overview、interactive demos、highlights、downstream、figures 都已经有清楚的 section。问题主要在视觉层级和颜色归属上，而不是结构本身。首页 hero 里确实是 Overview + “Multi-Omics Dynamical Integration”，后面再进入 Interactive、Highlights、Downstream、Figures。这个信息流可以保留。

### 1. Hero 区域：改成“白色论文封面 + 生物分子背景纹理”

现在 hero 里有绿色泡泡，标题渐变也有绿色，标签也是青绿，容易让开头过度偏绿。建议改成：

背景：白色玻璃卡片 + 很淡的青蓝/绿色径向光晕。
标题：主标题黑色，副关键词用蓝紫到青蓝渐变，不再用明显绿色。
标签：统一变成浅蓝灰底，左侧可以有一个很小的绿色点。
Figure 1 图片周围加轻微蓝灰投影，不要黄色或绿色边框。

建议改：

```css
.hero {

.hero-bg-bubbles span {

}
```

这样开头会变成“冷静、精密、生物计算”的感觉，而不是“绿色/黄色主题页”。

### 2. Kicker / Eyebrow / Card label：统一为同一种“小胶囊标签”

现在 `.eyebrow, .kicker, .card-label` 共用一套绿色/青色背景，但后面 `.card-label` 又被其他地方覆盖成普通标题样式，视觉语义不稳定。建议分开：

`.eyebrow, .kicker` 是 section 小标签；
`.card-label` 是卡片标题，不应该再用胶囊样式。

```css
.eyebrow,
.kicker {

  border: 1px solid rgba(63,111,216,.16);
  box-shadow: 0 6px 18px rgba(63,111,216,.08);

.card-label {
  color: var(--ink);
  background: transparent;
  border: 0;
  padding: 0;
  letter-spacing: 0;
  text-transform: none;
}
```

这样 Overview、Interactive、Highlights、Downstream 的开头会有统一风格，不再一会儿绿、一会儿黄、一会儿黑。

### 3. 大背景：保留“浅黄”，但把它变成科研纸色

你说“背景的黄色能不能再加一点，显得更生物、更科研型”。我的建议是：**不是加黄，而是加非常淡的暖黄 + 冷青灰纹理**。纯加黄会更像宣传页，不像 Nature Methods；加一层极淡网格/细胞光晕，会更科研。

```css
body {
```

## 五、JS 里的颜色也要改，否则 CSS 改完仍然乱

现在 JS 中数据轨迹颜色有两套：`TRAJ_COLORS = ["#2F2F85", "#CC4A77", "#FFC829"]`，以及 `TRAJ_4COLORS = ["#56337A", "#C24A6E", "#F0B537", "#7ABC3C"]`。这里黄色和绿色都是高饱和数据色，如果页面背景也偏黄/绿，就会冲突。

建议把数据渐变调成“深蓝 → 紫红 → 琥珀”，绿色只保留给生物 growth 或 cell-type：

```js
const TRAJ_COLORS = ["#25346F", "#7B4CC2", "#D89B2B"];

const TRAJ_4COLORS = ["#25346F", "#5E63D6", "#B65683", "#D89B2B"];
```

如果一定要保留“早→晚”里最后一个绿色，也要降饱和：

```js
const TRAJ_4COLORS = ["#25346F", "#5E63D6", "#B65683", "#7CBF9A"];
```

我更推荐第一版，因为黄色/琥珀更适合表达“time progression”，绿色留给 biological growth/proliferation。

## 六、实施顺序

第一步，先改 CSS 变量。不要到处单独改颜色，先把 `:root` 变成完整 design tokens。这样后续所有组件自动跟着统一。

第二步，改全局背景和 hero。这里决定页面第一眼的高级感。把黄色背景变成 warm ivory，把绿色泡泡降透明度，把标题渐变从“青-紫-绿”改成“青-紫”。

第三步，统一所有 section 小标题。`.eyebrow` 和 `.kicker` 保持胶囊；`.card-label` 从胶囊系统里拆出来，变成卡片标题。

第四步，统一卡片。`demo-card / concept-card / downstream-card / paper-figure-card` 全部用同一种白色玻璃卡片，不要一个偏灰、一个偏绿、一个偏黄。

第五步，调整 JS 色板。轨迹颜色、TF legend、cell type 颜色与 UI 颜色分离。UI 负责网页品牌感；JS 数据色负责科学图的语义。

第六步，看四个检查点：

1. 页面缩小到只看色块时，是否只有“暖白、深蓝、青蓝、少量绿/琥珀”；
2. 所有按钮是否都是同一主色；
3. 黄色是否只出现在数据/时间/扰动语义里；
4. 绿色是否只作为生物辅助，而不是到处当标题色和背景色。

## 七、我建议最终采用的设计口径

不要做成“绿色生物网站”，也不要做成“黄色温暖网站”。MOFI 是多组学动态推断框架，最佳气质应该是：

**白底论文感 + 蓝紫计算感 + 青绿生物感 + 琥珀时间动态感。**

这样颜色就有明确分工：


| 颜色         | 用途                    | 使用比例 |
| ------------ | ----------------------- | -------: |
| 暖白 / ivory | 页面背景、hero 纸感     |  55–65% |
| 深蓝黑       | 正文、标题、边界层级    |  20–25% |
| 青蓝/蓝紫    | 按钮、导航、主要强调    |   8–12% |
| 生物绿       | 小圆点、growth、bio cue |    3–5% |
| 琥珀黄       | 时间、轨迹、扰动强度    |    2–4% |

这个比例一稳定，页面会马上从“颜色堆叠”变成“有设计系统的科研网站”。
